Compiling Envision from source

Envision is written in C++ and uses features from the C++11 standard.

Make sure your C++ compiler is up-to-date. GCC version 4.8 or newer should be sufficient. Other dependencies are:

  • Qt 4.8: http://qt-project.org/
    This is the underlying framework that we use to build Envision's visualizations and cross-platform support. A migration to Qt 5 is planned in the near future.
  • lp_solve 5.5: http://lpsolve.sourceforge.net/
    Lp Solve is used for calculating some of the more complex visualization layouts.
  • Clang 3.4: http://clang.llvm.org/
    Clang is used for importing C++ projects in Envision. Support for importing C++ is preliminary and incomplete.
  • On Linux: Google Performance Tools: http://code.google.com/p/gperftools/
    Used for profiling.

To install the dependencies on Ubuntu 13.10, do the following:

  1. Install the Qt SDK: 
    sudo apt-get install qt-sdk
  2. Install the lp_solve development library version 5.5:
    1. Install the packages lp-solve (for the shared library) and liblpsolve55-dev (for the header files): 
      sudo apt-get install lp-solve liblpsolve55-dev
    2. Make the following symbolic link: 
      sudo ln -s /usr/lib/lp_solve/liblpsolve55.so /usr/lib/liblpsolve55.so
  3. Install Clang 3.4: 
    sudo apt-get install libclang-3.4-dev clang-3.4 llvm-3.4
  4. Install the Google Performance Tools: 
    sudo apt-get install google-perftools libgoogle-perftools-dev

Envision's repository contains configuration files for both Qt Creator and Eclipse. You can choose which tool you want to use but we recommend Qt Creator. The instructions below show you how to set-up both IDEs.

Qt Creator (recommended for C++; does not work with Java)

  1. Clone Envision's GitHub repository to a directory on your computer. Have in mind that during the build process Qt Creator will create additional directories alongside the repository clone.
  2. Start Qt Creator and configure the version of Qt that you want to use. You can leave the default settings or you can point to a custom Qt version. See below.
  3. Open the envision.pro file with Qt Creator.
  4. Configure the license template (Tools > Options > C++ > File Naming) to point to the LICENSE file inside the main repository directory.

Eclipse (recommended for Java; works for C++, but could require a lot of tweaking)

  1. Install the latest version of Eclipse.
  2. Download and install the Qt Integration plug-in for Eclipse. This plug-in is, unfortunately, no longer supported and the latest version is quite outdated. Look for the file: qt-eclipse-integration-linux.x86_64-1.6.1.tar.gz
  3. Start Eclipse and configure the version of Qt that you want to use. You can use the version provided by your system or you can point to a custom Qt version. See below.
  4. Clone Envision's GitHub repository to a directory on your computer. The project is structured in separate directories where each one is an Eclipse project.
  5. Import all projects into your Eclipse workspace as Git projects. Eclipse/Qt might try to change some project files. Reject all requested changes.
  6. Configure Eclipse's C++ file templates to use the text from the LICENSE file from the main directory in the repository.

Using a custom build of Qt

In order to make Envision's visualizations scale to very large projects we modified the rendering process in the Graphics View framework to avoid rendering objects which are too small. This customized Qt version can be found here: https://qt.gitorious.org/qt/qt-envision-enhancements

If you want to use Envision with a large amount of visual objects displayed at once we recommend you build and use the above version of Qt. Here is how:

  1. Clone the above repository into a new directory called qt-envision-enhancements inside the qt directory of the main Envision repository. The qt directory is ignored so you don't have to worry about polluting the main repository.
  2. Refer to the Qt's compilation guidelines to make sure that you have installed all dependencies. Here is a useful list for Ubuntu: Compile Qt 4.7 on Ubuntu 10.10
  3. Run the configure-envision script to confire the compilation of all components necessary for Envision.
  4. Run make and then make install .
  5. You should have a directory called qt-build alongside the qt-envision-enhancements directory.
  6. Make a symbolic link from DebugBuild/qt to qt/qt-build/lib .
  7. Configure Qt Creator or Eclipse to use the Qt found in qt-build .
